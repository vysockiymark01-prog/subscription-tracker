# Техзадание: PWA «Трекер подписок»

## Что делаем
PWA-приложение для учёта платных подписок с напоминаниями о списаниях. Рынок: Россия/СНГ. Язык интерфейса: русский. Позиционирование: приватность (никакого доступа к банкам, все данные локально) и простота (одна функция, сделанная хорошо).

Создай проект в текущей папке.

## Стек
- React + Vite
- PWA: manifest.json, service worker (vite-plugin-pwa)
- Хранение данных: localStorage через простую обёртку (позже заменим на IndexedDB, поэтому весь доступ к данным — только через один модуль `storage.js`)
- Уведомления: Web Push / Notification API (напоминания о списаниях)
- Стили: обычный CSS или CSS-модули, без Tailwind
- Никакого бэкенда в первой версии

## Модель данных
Подписка:
```
{
  id: string,
  name: string,            // «Яндекс Плюс»
  price: number,           // в рублях
  currency: "RUB",
  period: "month" | "year",
  nextPaymentDate: string, // ISO-дата следующего списания
  category: "video" | "music" | "software" | "games" | "education" | "other",
  isTrial: boolean,        // пробный период
  trialEndDate: string | null,
  status: "active" | "cancelled",
  cancelledAt: string | null,
  reminderDays: number,    // за сколько дней напомнить: 1, 3 или 7
  iconKey: string | null   // ключ пресета для иконки
}
```

После даты списания nextPaymentDate автоматически сдвигается на следующий период при открытии приложения.

## Каталог пресетов
Встроенный JSON-каталог популярных российских сервисов с типовыми ценами (пользователь может изменить цену при добавлении): Яндекс Плюс, Кинопоиск, VK Музыка, Озон Premium, СберПрайм, Telegram Premium, ЛитРес, Wink, Okko, IVI, START, Premier, МТС Premium, YouTube Premium, Spotify, iCloud, Netflix. Для каждого: название, типовая цена, категория, простая SVG-иконка или буквенная заглушка в цветном круге.

## Экраны

### 1. Главный экран
- Крупно сверху: «В месяц: X ₽» и «В год: Y ₽» (годовые подписки пересчитываются в месячный эквивалент и наоборот)
- Список активных подписок, отсортированный по ближайшей дате списания
- В каждой карточке: иконка, название, цена/период, «спишется через N дней»
- Подписки со списанием в ближайшие 3 дня подсвечены
- Кнопка «+» для добавления
- Если список пуст — приветственный экран с кнопкой «Добавить первую подписку»

### 2. Добавление подписки
- Сетка пресетов + поиск по ним
- Кнопка «Своя подписка» — пустая форма
- После выбора пресета: форма с предзаполненными полями (название, цена, категория), пользователь указывает дату следующего списания и период
- Переключатель «Пробный период» → появляется поле даты окончания триала
- Выбор напоминания: за 1 / 3 / 7 дней

### 3. Карточка подписки
- Все данные, редактирование
- Кнопка «Отменил подписку» → статус cancelled, уходит в архив
- Кнопка «Удалить» (с подтверждением)

### 4. Архив
- Отменённые подписки
- Сверху: «Ты экономишь X ₽ в год» (сумма отменённых в годовом выражении)
- Возможность вернуть подписку в активные

### 5. Статистика (простая)
- Разбивка расходов по категориям (горизонтальные полосы, без библиотек графиков)
- Топ-3 самых дорогих подписки в годовом выражении

## Напоминания
- При добавлении первой подписки запросить разрешение на уведомления (с объяснением зачем)
- Service worker проверяет при открытии приложения ближайшие списания; локальное уведомление за reminderDays до даты
- Для триалов — отдельное напоминание «Пробный период заканчивается, отмени, если не нужно»
- Если разрешение не дано — показывать бейдж-предупреждение внутри приложения

## Дизайн
- Mobile-first, максимальная ширина контента 480px, на десктопе — по центру
- Тёмная и светлая тема по системной настройке
- Минимализм: одна акцентная цветовая переменная, крупная типографика для сумм
- Нижняя навигация: Главная / Статистика / Архив / Настройки

## Настройки
- Тема (системная/светлая/тёмная)
- Экспорт данных в JSON-файл и импорт из него (страховка пользователя от потери данных)
- Ссылка «О приложении»

## Что НЕ делать в этой версии
- Никаких интеграций с банками и парсинга СМС
- Никакой авторизации и серверной синхронизации
- Никаких библиотек графиков, никакого Redux — состояние через useState/useContext
- Никакой оплаты — монетизацию добавим позже
